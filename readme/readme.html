简要介绍

FGTK - Fat GC-MS ToolKit是用shiny与一系列R包开发的应用，主要用于解决安捷伦气质联用中老旧仪器（如：Agilent 6890N与Agilent 5975C）的数据分析问题，安捷伦较新仪器可以用MassHunter处理一批数据，但老旧仪器只能用MSD Chemstation做一些简单的数据前处理。FGTK可以根据MSD Chemstation导出的结果进行下游分析。

流程


以上是该软件大致的使用流程。

注意

本应用由东北林业大学林学院森林生态系统可持续经营教育部重点实验室开发。

特别注意

FGTK只需要你提供文件夹路径，就可以进行各个操作，目前不提供其他数据输入方法。FGTK会自动识别该文件夹下功能所需的文件，如果没有会提醒你！！！
你只需要单机选中某个文件夹，然后点击选择文件夹。
选择文件夹第一次时需要您信任新生成的ps文件，否则无法弹出窗口让你选择文件夹！！！只有信任才可以！！！

MSD chemstation

xls文件需要以上流程，需要多个报告导出为xls，其中至少要勾选积分结果与谱库检索结果。该文件包含主要功能所需的数据。
csv文件需要以上流程，需要导出数据至CSV文件，选择多个文件-带有数据文件，选择RO中的色谱图，选择否，选择需要的数据，之后点执行。该文件包含色谱图数据，是2D TIC与3D TIC功能所必须的。

xls文件里包含你的积分结果与谱库检索结果，是大多数功能所必须的。而CSV文件是TIC数据是2D TIC与3D TIC功能所必须的。这两种文件需要你分开存放！！！

Migration

MSD Chemstation导出的多个结果与TIC数据会放在每个data.D文件夹里，Migration会将其中的xls文件或CSV文件复制到目标文件夹(migrated data folder path)中并以文件夹的名称重命名。你只需提供存储data.D文件夹的父文件夹路径(raw data folder path)。
之后需要您重新整理xls文件或CSV文件，将不同处理的xls文件或CSV文件放在不同的文件夹里。不然无法运行之后的功能！！！子目录的名称会成为之后分析里的group！！！另外xls文件与CSV文件请分开放！！！

Compare GC-MS results

Compare GC-MS results里的各个参数需要按照推荐的范围来设置，不然可能会导致运行失败或者无法结束程序！！！
Compare GC-MS results功能里的input folder path是使用Migration功能后并整理后的文件夹路径，output folder path是结果存储的文件夹路径。
本应用其余功能的需要路径主要多为compare result folder，这也是Compare GC-MS results功能里的output folder path。

Compare GC-MS results运行完后需要您亲自调整，可能有一个物质分成了两行等情况出现，这主要是因为含量差距过大，导致质谱数据或者保留时间相差过大，难以合并在一起。
另外可能会出现多个峰判定为同一物质，需要你自己判断，并删除错误的峰！！！

Filter

参考自非靶代谢组数据处理，主要用于筛选化合物，删除掉某些过于少的化合物，默认保留总体样本里出现频率大于0.2或者某个类别里出现频率大于0.4的化合物。

另外也会删除对照组里的杂质。对照组默认文件夹名称为CK。
同时运行该功能后，原来的peak.xlsx变为peak_no_filtered.xlsx，而filter后的peak.xlsx取代原来的peak.xlsx。sample_info.xlsx文件也是如此。

Pubchem

一般来说，100个化合物会运行5-10分钟，根据网络状态而不同。
该功能需要联网，且运行时间长，请耐心等待！！！
由于NIST版本问题，可能出现Hitname与Pubchem网站上名称不同的情况，需要您亲自在函数运行完后检查compound_info.xlsx文件，把Hitname改成正确名称，重新运行该功能或者直接填补缺失的信息。
该功能运行后请务必完成上述操作，否则会影响Classyfire功能的运行！！！

Classyfire 与 NP Classifier

该两个功能需要联网，且运行时间长，请耐心等待！！！
一般来说，Classyfire分析100个化合物会运行5-10分钟，NP Classifier分析100个化合物会运行1-2分钟根据网络状态而不同。Classyfire与NP Classifier得到结果不同，请结合两者一同分析物质。Classyfire分类标准与主流代谢组数据库相近，如果出现分歧以第一个为准。

Classification summaries

该功能主要是将化合物的分类结果进行总结，依据不同样本不同类别进行计算与计数。
该功能需要compound_info.xlsx文件，同时需要你另添加一个Class列，其中是你的分类结果，可以是Classyfire与NP Classifier的结果，也可以是你自己分类的结果。

Stat

该功能主要计算相对含量与进行单因素方差分析。

Multivariate data analysis

该功能主要进行PCA、PCoA与NMDS分析与可视化。

Biomarker

该功能会在compare result folder文件夹下生成biomarker子目录，其中会有各种group之间的差异比较（t检验与火山图）、PLS-DA与OPLS-DA，具体文件见下图。
如果有部分文件缺失，则表明该数据无法运行各种方法！！！另外就算有各种文件，也需观察具体内容，查看各个分析对应的要求！！！

Plots

该模块由Venn、Upset、Cluster、2D TIC与3D TIC组成，进行主要数据分析外的一些图片绘制，更改参数后需要你重新点plot按钮。
Cluster与2D TIC提供另外的分析结果的下载。
2D TIC与3D TIC功能需要TIC数据，提取该数据流程参考MSD chemstation小节。
